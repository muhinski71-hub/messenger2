# Мессенджер (как Discord)

Веб-мессенджер на **FastAPI + WebSocket** с текстовым чатом в реальном времени и голосовым чатом (WebRTC).

## Возможности
- 💬 Текстовый чат в реальном времени
- 🎙 Голосовой чат (WebRTC, peer-to-peer mesh)
- 🏠 Комнаты (каналы) — вводишь название комнаты при входе
- 👥 Список участников с индикатором «в голосовом»
- 🔇 Кнопка выключения микрофона
- 📜 История последних сообщений в комнате

## Запуск локально

```bash
cd messenger
python -m pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

Открой в браузере: http://localhost:8000

Чтобы друзья в одной Wi-Fi сети подключились — дай им твой локальный IP, например `http://192.168.1.50:8000`.

## Важно про голосовой чат

Браузер разрешает доступ к микрофону только на **`localhost`** или по **HTTPS**.
Поэтому для общения через интернет нужен домен с HTTPS (см. ниже).

## Деплой в интернет

Подойдёт любой хостинг с поддержкой WebSocket и HTTPS. Проще всего:

### Вариант 1: Render.com / Railway.app (бесплатно)
1. Залей папку `messenger` в репозиторий GitHub.
2. Создай новый **Web Service**, укажи:
   - Build: `pip install -r requirements.txt`
   - Start: `uvicorn main:app --host 0.0.0.0 --port $PORT`
3. Платформа сама выдаст HTTPS-домен — голосовой чат заработает.

### Вариант 2: Docker
```bash
docker build -t messenger .
docker run -p 8000:8000 messenger
```

## Как пользоваться
1. Открой сайт, введи имя и название комнаты (одинаковое у друзей).
2. Пиши сообщения в чат.
3. Нажми **«Войти в голосовой»**, разреши доступ к микрофону — и говори с теми, кто тоже в голосовом.

## Заметки
- История сообщений хранится в памяти сервера (сбрасывается при перезапуске). Для постоянного хранения можно подключить БД.
- Голосовой чат — mesh P2P, оптимален для небольших групп (до ~6-8 человек). Для больших групп нужен SFU-сервер.
